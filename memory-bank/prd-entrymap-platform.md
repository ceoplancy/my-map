# 앤트리맵(EntryMap) SaaS 플랫폼화 PRD

**프로젝트**: entre-map → B2B SaaS 플랫폼  
**버전**: 0.1 (요구사항 정의)  
**브랜치**: dev (staging)

---

## 1. 프로젝트 개요

기존 **단일 회사용** 주주·주식 관리 시스템을 **B2B SaaS 플랫폼**으로 고도화한다.  
의결권 대행사 및 상장사가 가입 후 자신만의 워크스페이스(프로젝트 맵)를 만들고, 주주명부와 현장 요원(용역)을 독립적으로 관리할 수 있도록 한다.

---

## 2. 즉시 반영 요청 사항 (단기)

- **주주명부 활성화 기간**: 주주명부 **최초 생성 시** 활성화 기간 설정  
  - 활성화 기간에 따른 지도 노출 여부·수정 가능 여부는 협의 중.
- **용역 사진 기능 확장**:  
  - 현재: 신분증 사진 수신 등  
  - **추가**: 용역이 **직접 사진 촬영**할 수 있는 기능 (현장에서 바로 촬영 권장, 수정 이력만 남기는 용도 고려)

---

## 3. 플랫폼 진입 및 계정 관리 (B2B)

- **진입**: 복잡한 랜딩 없이 **로그인/회원가입** 중심의 직관적 화면.
- **회원가입·승인**  
  - 가입 시 **[상장사] / [의결권 대행사]** 중 선택 (동일 플랫폼 UI).  
  - 가입 즉시 사용 불가 → **가입 대기**.  
  - **앤트리맵 최고 관리자(운영사)**가 확인 후 승인/반려하여 품질 통제.
- **용역 충원 요청**  
  - 우측 상단 **[용역 요청]** 버튼 상시 노출.  
  - 클릭 시 운영사에게 알림 + 이메일 전송.

---

## 4. 대행사/상장사 관리자 패널 (워크스페이스)

### 4.1 주주명부 업로드 및 오류 수정

- **복수 명부 관리**: 한 계정에서 **여러 상장사** 주주명부 동시 업로드·관리.
- **활성화 기간**: 주주명부 최초 생성 시 **활성화 기간** 설정 (노출/수정 규칙은 협의 중).
- **주소 클렌징 UX**  
  - 기존: 실패 건 다운로드 → 수정 → 재업로드.  
  - 개선: **성공 건은 바로 적용**, 지도 맵핑 **실패 건은 화면 내 '오류 리스트'**에 표기.  
  - 관리자가 해당 화면에서 **주소 텍스트 직접 수정** → 즉시 반영.
- **주식 수 기준**: 대상 타겟의 주식 수 기준(예: 특정 주식 수 이상) 파악이 쉽도록 구성.

### 4.2 맵 노출 관리 및 대시보드

- **노출 토글 (ON/OFF)**: 의결권 모집 마감 또는 작업 중지 시, 주주명부별로 **앤트리맵 노출/비노출**을 한 번에 제어하는 스위치.
- **현황 대시보드**: 복잡한 그래프 지양. 상장사별/전체 기준 **[완료 / 보류 / 실패 / 미방문]**의 **인원 수·주식 수**를 텍스트/숫자로 심플하게 요약.
- **통합 필터 (AND 조건)**: 상태별·소속별·특정 인원별 등 **다중 중첩 필터** → 대시보드 요약 수치 실시간 연동.  
  - 필터 적용 리스트 **엑셀 내보내기** 지원.

### 4.3 용역 인원 및 이력

- **독립 용역 세팅**: 관리자가 **용역 계정** 생성 (이메일, 비밀번호, 이름, 소속), **권한(일반/팀장)** 및 **담당 상장사** 할당.
- **상세 변경 이력(History)**: 누가, 언제, 어떤 상태/메모를 변경했는지 **서버에 기록·추적**.

---

## 5. 현장 요원(용역) 모바일 패널

원칙: **모바일에서의 속도·안정성·가독성** 우선. 기능 제약은 완화하고 디자인 자유도 확보.

### 5.1 지도 마커 및 상세 팝업

- **마커 상태 색상**: 🔴 미방문(빨강) / 🟡 보류(노랑) / 🟢 완료(초록) / ⚫ 실패(검정).
- **길찾기**:  
  - 주주 상세 팝업에 **[카카오맵/네이버지도로 길찾기]** 딥링크 버튼 권장.  
  - 공수·이슈 시 기존 **주소 복사** 유지 가능.

### 5.2 상태 변경 및 이력

- **상태·메모**: 팝업에서 상태 변경·메모 작성 가능.
- **사진 업로드**  
  - 위임장/신분증: **모바일 카메라 촬영** 또는 **앨범 첨부**.  
  - **주주별 사진 업로드 링크**: 요원이 직접 찍기를 부담스러워하는 경우를 위해, **주주별로 사용자가 직접 찍어 올릴 수 있는 링크**(및 **QR 코드**) 표시 → 현장에서 해당 링크/QR로 업로드 유도.  
  - **활동 사진**: 각 용역 인원이 **활동 사진** 업로드 가능. **바로 촬영해서 업로드**만 지원 (사진첩 선택 불가).

- **이력(History) 뷰**: 기존 변경 이력 데이터를 **모바일에서 글씨가 깨지지 않고 깔끔하게** 조회 가능하도록 UI 개편 (형태는 디자이너 재량).

### 5.3 필터 및 보안

- **요원용 다중 필터**: 상장사별·지역별·상태별 마커 필터.  
  - **주식 수 구간 필터** 다중 선택 (예: 1천 미만 ~ 60만 이상).  
  - UI는 모바일에 맞게 자유롭게 구성.
- **보안 (선택)**: 엑셀 다운로드 차단·화면 캡처 방지는 **이번 페이즈 필수 아님**, 상황에 맞게 탄력 적용.

---

## 6. 기획·개발 범위 관련

- 기획이 부족한 부분은 **질문 시 바로 답변·확인** 요청.
- **전체 내용을 모두 이번에 개발하지 않는다**는 점을 클라이언트에 공유한 상태. 우선순위·페이즈에 따라 단계적 적용 예정.
